# 営業日報システム 要件定義書

## 1. システム概要

営業担当者が日々の訪問活動を報告し、上長がフィードバックを行うためのWebシステム。

## 2. ユーザー種別

| 種別 | 説明 |
|------|------|
| 営業担当者 | 日報を作成・編集する |
| 上長（マネージャー） | 日報を閲覧し、コメントする |

## 3. 機能要件

### 3.1 日報管理機能

#### 日報作成・編集
- 営業担当者は1日1件の日報を作成できる
- 日報には以下の情報を含む：
  - 日付
  - 訪問記録（複数件登録可能）
  - Problem（課題・相談事項）
  - Plan（明日やること）

#### 訪問記録
- 1つの日報に複数の訪問記録を追加可能
- 各訪問記録には以下を含む：
  - 訪問先顧客（顧客マスタから選択）
  - 訪問内容（テキスト）
  - 訪問時間（任意）

### 3.2 コメント機能

- 上長は日報のProblem/Planに対してコメントを追加できる
- コメントは複数件追加可能
- コメント対象を指定可能（Problem or Plan）

### 3.3 マスタ管理機能

#### 顧客マスタ
- 顧客情報の登録・編集・削除
- 顧客名、住所、電話番号、担当者名等を管理

#### 営業マスタ（ユーザー管理）
- 営業担当者の登録・編集・削除
- 氏名、メールアドレス、所属部署、役職（営業/上長）を管理

### 3.4 閲覧機能

- 日報一覧表示（日付、担当者でフィルタ可能）
- 日報詳細表示
- 自分の過去日報の参照

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| 対応ブラウザ | Chrome, Safari, Edge（最新版） |
| レスポンシブ | PC・タブレット対応 |
| 認証 | ログイン機能（メール/パスワード） |

## 5. 画面一覧

| 画面名 | 説明 |
|--------|------|
| ログイン | ユーザー認証 |
| 日報一覧 | 日報のリスト表示 |
| 日報作成・編集 | 日報の入力フォーム |
| 日報詳細 | 日報内容とコメント表示 |
| 顧客マスタ | 顧客の一覧・登録・編集 |
| ユーザー管理 | 営業担当者の一覧・登録・編集 |

## 6. データ項目

### 日報（DailyReport）
- ID
- 営業担当者ID
- 日付
- Problem（課題・相談）
- Plan（明日やること）
- 作成日時
- 更新日時

### 訪問記録（Visit）
- ID
- 日報ID
- 顧客ID
- 訪問内容
- 訪問時間（任意）
- 作成日時

### コメント（Comment）
- ID
- 日報ID
- 投稿者ID（上長）
- 対象区分（Problem/Plan）
- コメント内容
- 作成日時

### 顧客（Customer）
- ID
- 顧客名
- 住所
- 電話番号
- 担当者名
- 作成日時
- 更新日時

### ユーザー（User）
- ID
- 氏名
- メールアドレス
- パスワード（ハッシュ）
- 部署
- 役職（営業/上長）
- 作成日時
- 更新日時
